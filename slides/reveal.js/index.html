Was ist eine Javascript Engine?

A JavaScript engine is a program or interpreter which executes JavaScript code. A JavaScript engine may be a traditional interpreter, or it may utilize just-in-time compilation to bytecode in some manner. Although there are several uses for a JavaScript engine, it is most commonly used in Web browsers.

Welche Javascript Engines gibt es?

  • V8: Google Chrome & NodeJS
  • Chakra: Microsoft Browsers
  • Rhino, Nashorn: Java
  • SpiderMonkey: Netscape, Firefox
  • JavascriptCore: Apple

Warum Typescript?

Typescript Entwicklung ist die effizientere JavaScript Entwicklung.

  • Compiler findet Syntaxfehler.
  • Typ-Prüfung vom Compiler.
  • d.ts Typen Definitonen für existierende Javascript Libraries.

Kompilierung

JavaScript ist eine interpretierte Sprache. Deshalb muss sie erst ausgeführt werden, bevor sie auf Gültigkeit geprüft werden kann. Man schreibt den Code also und schaut dann, ob man eine Ausgabe bekommt. Wenn keine Ausgabe angezeigt wird, ist ein Fehler aufgetreten. Also verbraucht man viel Zeit einen Fehler zu finden. Der TypeScript Transpiler bietet ein Feature zur Fehlerprüfung. TypeScript compiliert den Code und wirft Kompilierungsfehler wenn es inene Syntaxfehler findet. Somit werden Fehler erkannt, bevor das Skript ausgeführt wird.

Strong Static Typing

JavaScript ist nicht streng typisiert. TypeScript stellt eine optionale statische Typisierung und ein Typenreferenzsystem durch das TLS (TypeScript Language Service) zur Verfügung. Der TLS fügt einem automatische einen Typ zu, wenn kein Typ definiert wird.

Typdefinitionen

TypeScript unterstützt Typdefinitionen existierende, externe JavaScript Bibliotheken. Somit kann TypeScript code diese Bibliotheken einbinden.

Objektorientierte Programmierung

TypeScript unterstützt Konzepte der objektorientierten Programmierung, wie z.B. Interfaces und Vererbung.
